Enterprise engineering review

Fenic turns semantic data work into inspectable DataFrame pipelines

Fenic is a Python DataFrame engine for structured and unstructured data where LLM calls, embeddings, extraction, classification, summarization, semantic joins, and clustering are operators in the query plan rather than side scripts.

Installed package: fenic 0.10.0 12 local operator scripts executed Self-contained HTML

Executive Summary

For enterprise engineers, Fenic is best understood as semantic DataFrames for reusable LLM and data pipelines. The project README describes it as a semantic DataFrame engine with PySpark/SQL-style operations plus AI operators such as extract, classify, summarize, embed, semantic joins, lazy plans, session-level model configuration, batching, rate limiting, retries, token and cost accounting, caching, lineage, and query metrics. Source: typedef-ai/fenic README.

The practical value is not that Fenic removes probabilistic behavior from LLMs. It puts that behavior inside typed, inspectable, rerunnable pipelines that can be reviewed, cached, promoted, and exposed to agents or downstream systems.

Running these examples may create local .duckdb artifacts and may incur OpenAI API costs through OPENAI_API_KEY. Treat local caches, databases, prompts, input rows, and model outputs as sensitive unless your data classification says otherwise.

Where It Fits

Support-ticket triageExtract products, severity, sentiment, root cause, and routing fields from messy customer text.
Document extractionParse PDFs or text-heavy documents into typed rows for review, search, and workflow automation.
Eval analysisClassify agent traces, summarize failures, and rerun the same analysis against new model versions.
Semantic joinsMatch records by meaning when exact keys do not exist, such as incidents to policies or resumes to roles.
Summarization pipelinesReduce many rows into concise group summaries after filtering and joining with structured metadata.
Agent-facing data toolsPromote tables, views, or parameterized tools so agents call governed data surfaces instead of improvising queries.

Pros and Cons

Strengths

  • Typed semantic operators make LLM calls reviewable parts of a query plan.
  • Lazy, DataFrame-style pipelines compose with ordinary filtering, joining, grouping, ordering, and SQL.
  • Model configuration lives at the session level, with provider support for language and embedding models.
  • Batching, rate limiting, retries, token and cost accounting, response caching, lineage, and metrics improve operability.
  • fenic check gives agent-written Fenic code a static API sanity check before execution.

Risks and Friction

  • Local .duckdb files and caches create an artifact trail that teams must govern.
  • In the installed Fenic package, lancedb is a required dependency, and the current examples/configuration do not expose vector database selection as a user-facing option.
  • API usage can scale cost quickly if broad semantic operations run without sampling, caching, and budgets.
  • LLM outputs remain probabilistic and need evals, guardrails, and human review for high-impact flows.
  • The project is young enough that adoption, API stability, and operational playbooks should be validated through pilots.

Enterprise Readiness Matrix

ScalabilityPromising for batched, repeatable semantic processing, but validate throughput against provider limits, large document sets, and local DuckDB/LanceDB behavior.
SecurityTreat prompts, source rows, outputs, caches, and local database files as sensitive data; enforce provider data-handling policy review before production use.

Cost ControlSession-level model configuration, batching, caching, and token/cost metrics are useful controls; teams still need budgets, sampling, and query review for broad runs.
ObservabilityFenic exposes plans, lineage, metrics, and rerunnable pipelines, which is materially better than notebook-only prompt chains.
ReproducibilityLazy typed pipelines and response caching improve repeatability, while probabilistic model behavior still requires evals and pinned model versions.
MaintainabilityReusable DataFrame pipelines are easier to review than scattered ad hoc LLM calls; API maturity and team familiarity remain adoption risks.
Integration FitBest fit for Python data/ML/analytics teams and agent tooling surfaces; less natural if the enterprise standardizes all processing inside a warehouse engine.
Operational MaturityViable for pilots and controlled internal workflows today; production use should start behind careful data classification, cost guardrails, and artifact hygiene.

Shared session configuration

common.py is excluded from the numbered gallery because it is shared setup, but it defines the OpenAI language and embedding model aliases used by the examples.

from __future__ import annotations

import fenic as fc


def get_session(app_name: str) -> fc.Session:
    config = fc.SessionConfig(
        app_name=app_name,
        semantic=fc.SemanticConfig(
            language_models={
                "mini": fc.OpenAILanguageModel(
                    model_name="gpt-5.5",
                    rpm=100,
                    tpm=100_000,
                )
            },
            default_language_model="mini",
            embedding_models={
                "small": fc.OpenAIEmbeddingModel(
                    model_name="text-embedding-3-small",
                    rpm=100,
                    tpm=100_000,
                )
            },
            default_embedding_model="small",
        ),
    )
    return fc.Session.get_or_create(config)

from __future__ import annotations

from pydantic import BaseModel, Field

import fenic as fc
from common import get_session


class Ticket(BaseModel):
    product: str = Field(description="Product or feature area mentioned by the user")
    sentiment: str = Field(description="positive, neutral, or negative")
    issue: str = Field(description="Short description of the issue or praise")


session = get_session("fenic_operator_extract")

df = session.create_dataframe(
    [
        {"id": 1, "text": "The CSV export in Reports times out every morning."},
        {"id": 2, "text": "The new dashboard is fast and much easier to read."},
    ]
)

result = df.select("id", fc.semantic.extract("text", Ticket).alias("ticket")).unnest("ticket")

result.show()

    [
        {
            "id": 1,

                "shows whether the import succeeded or failed. I have retried three times and "
                "now I am worried we may have created duplicate records."

        },
    ]
)

result = df.select(
    "id",
    fc.semantic.map(

        text=fc.col("text"),

)

import fenic as fc

        column=fc.col("note"),
    ).alias("theme")

from __future__ import annotations

from pathlib import Path

import fenic as fc
from common import get_session


def write_minimal_pdf(path: Path) -> None:
    path.parent.mkdir(parents=True, exist_ok=True)
    path.write_bytes(
        b"%PDF-1.4\n"
        b"1 0 obj<</Type/Catalog/Pages 2 0 R>>endobj\n"
        b"2 0 obj<</Type/Pages/Count 1/Kids[3 0 R]>>endobj\n"
        b"3 0 obj<</Type/Page/Parent 2 0 R/MediaBox[0 0 612 792]/Contents 4 0 R/Resources<</Font<</F1 5 0 R>>>>>>endobj\n"
        b"4 0 obj<</Length 68>>stream\nBT /F1 18 Tf 72 720 Td (Fenic parses PDF files into Markdown.) Tj ET\nendstream\nendobj\n"
        b"5 0 obj<</Type/Font/Subtype/Type1/BaseFont/Helvetica>>endobj\n"
        b"xref\n0 6\n0000000000 65535 f \n0000000009 00000 n \n0000000056 00000 n \n"
        b"0000000111 00000 n \n0000000223 00000 n \n0000000341 00000 n \n"
        b"trailer<</Size 6/Root 1 0 R>>\nstartxref\n411\n%%EOF\n"
    )


session = get_session("fenic_operator_parse_pdf")
pdf_path = Path(__file__).with_name("sample_parse_pdf.pdf")
markdown_path = pdf_path.with_suffix(".md")
write_minimal_pdf(pdf_path)

df = session.create_dataframe([{"id": 1, "pdf_path": str(pdf_path)}])

result = df.select(
    "id",
    fc.semantic.parse_pdf("pdf_path", page_separator="\n\n--- page {page} ---\n\n").alias("markdown"),
)

rows = result.to_pylist()
markdown = rows[0]["markdown"]

print(markdown)
markdown_path.write_text(markdown, encoding="utf-8")
print(f"\nWrote parsed markdown to {markdown_path}")
